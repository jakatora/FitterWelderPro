## P0 — Ship-blocker / data loss / crash risk

### P0-01 — Guard weld journal load/save/delete against disposed-state crashes
- **What**: Add `if (!mounted) return;` before every `setState` after an `await` in `_load`, `_cycleStatus`, `_delete`, `_save`, `_openEditor`, date-picker callback; wrap `_dao.*` calls in try/catch resetting `_loading`/`_saving` to false.
- **Where**: `lib/screens/weld_journal_screen.dart:193-199, 352-376, 573-604, 707-726`
- **Why**: Fast tap-back after opening the journal crashes the app mid-shift; save failures leave the editor wedged at "Zapisywanie…" and the welder loses 30 s of typed traceability data with no draft.
- **Effort**: M
- **Refs**: [3]

### P0-02 — Stop swallowing real SQLite errors in `_ensureTable` migrations
- **What**: Inspect the exception message; only `catch` "duplicate column"/"already exists"; re-throw locked-DB and disk-IO errors so the outer caller can surface a real error to the user.
- **Where**: `lib/screens/weld_journal_screen.dart:112-148`
- **Why**: A partially-failed migration after an app update silently corrupts every new weld save and cascades into the wedged "Zapisywanie…" state above.
- **Effort**: M
- **Refs**: [3]

### P0-03 — Block / warn on negative or out-of-range CUT values in cut list totals
- **What**: In `_totalCutMm` skip non-finite or negative `cutMm` and render a top-of-list warning chip "X segments excluded — invalid CUT"; clamp AI-suggested `dimensionMm` to `0 < v < 100000` before assigning to a segment.
- **Where**: `lib/screens/iso_scanner_screen.dart:69-83, 244-276, 572-579`
- **Why**: A typo like `-100` in a deduct or an AI hallucination silently corrupts the grand total — welder cuts pipe stock by metres less than needed; real scrap cost.
- **Effort**: S
- **Refs**: [2]

### P0-04 — Restore previous device id / add grace period before stripping PRO
- **What**: Before overwriting deviceId in `init()`, save the old id to `fitter_device_id_previous` and expose `restorePreviousDeviceId()`; in `refreshFromBackend()` keep `_previousStatus`, add a 2-minute grace window and a second confirm refresh before broadcasting an active→free downgrade.
- **Where**: `lib/services/premium_service.dart:107-117, 217-248`
- **Why**: A single tightened length check or a webhook race silently strips a paying customer's PRO mid-job with no recovery path — direct revenue + trust loss.
- **Effort**: M
- **Refs**: [50]

### P0-05 — Stop stale results from being copied after validation failure
- **What**: At the start of `_calculate()`, clear all four result controllers (or before any early-return guards) so a failed re-calc leaves the result fields blank instead of showing the previous job's number.
- **Where**: `lib/screens/rolling_offset_screen.dart:80-86`
- **Why**: Fitter computes Travel for pipe A, switches to pipe B, mis-types and hits CALCULATE — the stale "707.1" still sits in the field, gets copied, wrong-length weld.
- **Effort**: S
- **Refs**: [7, 32]
